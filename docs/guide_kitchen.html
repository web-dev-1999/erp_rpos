Introduction

The Kitchen Display System (KDS) is your real-time window into every order placed on the floor. As soon as a waiter sends an order from a table, it appears on the KDS as a ticket. You do not need to touch paper dockets, radio the floor, or wait for a printer — everything updates live.

Purpose

The KDS exists to give kitchen staff a clear, prioritised view of all incoming food orders, organised by preparation station. Your job is to read each ticket, start cooking when ready, and mark items done so the floor team knows exactly when to collect.

Accessing the KDS

Open any browser on the kitchen display terminal and navigate to:

http://[server]/desk/kds

Replace [server] with the local IP address or hostname provided by your manager (for example 192.168.1.10 or pos.restaurant.local).

ℹ
No personal login is required to use the KDS action screen. The display uses a shared kitchen session. However, the URL above must be loaded from an approved device on the restaurant network.

The KDS Screen Layout

The KDS screen is divided into vertical columns — one per preparation station. Common stations include Grill, Cold Kitchen, Desserts, Fryer, and Pastry. Your manager configures which stations appear.

Column Structure

  • Column header: Displays the station name (e.g. Grill) and a station-specific item count badge.
  • Tickets: Each card inside a column represents one table's order for that station. A table may have a ticket in multiple columns if the order contains items from different stations.
  • Chips: Each coloured pill inside a ticket is a single menu item. One chip = one ordered item.

Top Toolbar

  • Station Name — confirms which KDS view you are on (relevant when using multiple screens).
  • Refresh button — forces an immediate data reload (see Section 7).
  • Item count — total number of items currently in queue across all stations.

Chip Colour Reference

Sent — Not Started Cooking — In Progress Ready — Plated

Reading a Ticket

Each ticket card shows all the information you need at a glance. Here is what every element means:

Ticket Header

  • Table number — which table placed the order (e.g. T-07).
  • Covers — number of guests at the table (e.g. 4 covers). Useful context for portion sizing.
  • Time sent — the clock time the waiter transmitted the order (e.g. 19:42). Tickets are sorted oldest first so priority is always visible.

Chip Content

  • Item name — the menu item (e.g. Ribeye Steak 300g).
  • Quantity — number of portions (e.g. x2). If two identical items are ordered they appear as a single chip with quantity 2.
  • Notes / modifiers — any special instructions the waiter recorded (shown in smaller text beneath the item name).
T-07  ·  4 covers 19:42
Ribeye Steak 300g ×2 Medium rare — no sauce
Grilled Salmon ×1 Lemon butter on side
Seasonal Vegetables ×3  

Example ticket above: Table 7, 4 guests, order sent at 19:42. Two items still in progress, vegetables are ready.

⚠
Oldest first: Tickets near the top of a column have been waiting longest. Always work from the top of each column down to maintain correct service order.

Advancing Item Status

This is the main action you will take on the KDS. Each chip has a ▶ (play/advance) button on its right side. Tapping it moves the item forward one stage in the preparation lifecycle.

The Three Stages

Sent
→
Cooking
→
Ready
  • Sent → Cooking: Tap ▶ when you take the item from the queue and begin active preparation. The chip turns orange so the whole team can see it is in progress.
  • Cooking → Ready: Tap ▶ again when the dish is fully cooked and plated on the pass. The chip turns green.
  • All chips green: When every chip on a ticket reaches Ready status, the ticket fades out and is removed from the column automatically. The waiter receives a notification to collect the order.

Step-by-Step

  1. Locate the ticket for the table you are about to cook.
  2. Read all chip notes before touching any ingredient (see Section 6).
  3. Tap ▶ on each chip as you begin cooking that item (blue → orange).
  4. When the item is plated and on the pass, tap ▶ again (orange → green).
  5. If cooking multiple items simultaneously, advance each chip independently as they finish.
⚠
No confirmation step: Advancing a chip takes effect immediately. If you advance a chip by mistake, notify your manager — a void or correction must be done from the Order Screen. See Section 8 (Scenarios) for the recovery process.

5

Multi-Station Items

Some menu items are prepared across more than one station — for example, a plated dish that requires components from both the Grill and the Cold Kitchen. In this case the same ticket appears in both station columns, each showing only the chips relevant to that station.

How It Works

  • Each station sees and advances only its own chips.
  • You do not need to wait for another station to finish before advancing your own chips.
  • The system tracks readiness across all stations. The waiter is notified — and the ticket is fully dismissed — only once all items on the ticket are green across all stations.
ℹ
Coordination tip: If your chips are ready but the ticket has not disappeared, it means another station is still cooking items for that table. Communicate with the other station verbally so dishes do not sit on the pass and cool down.

Special Instructions

Special instructions entered by the waiter appear as a secondary line of text directly below the item name on each chip. These may include:

  • Allergy warnings — ALLERGY: No nuts
  • Cooking preferences — Well done, Medium rare
  • Modifications — No onions, Extra sauce on side
  • Guest preferences — Child portion, No salt
🚫
Always read the notes before you start cooking. Allergy instructions in particular must be followed exactly. If an instruction is unclear, ask your manager or the floor team before plating — never guess.

Long Notes

If a note is too long to fit on a single chip line, tap the chip once to expand it and see the full text. Tap again to collapse.

Refreshing the Queue

The KDS receives new orders in real time over the network. In most cases, tickets appear within a few seconds of the waiter sending them. However, if the connection hiccups or the browser tab has been open for a long time, the display may fall behind.

Auto-Refresh

The KDS automatically polls the server every 30 seconds. A small loading indicator flashes briefly in the toolbar each time this occurs. You do not need to do anything for this to work.

Manual Refresh

  1. Look for the Refresh button in the top toolbar (circular arrow icon).
  2. Tap it once to force an immediate reload of all pending tickets.
  3. The screen will briefly go blank, then repopulate with the current queue.
⚠
If the screen goes blank after a manual refresh and does not repopulate within 10 seconds, check that the kitchen terminal has an active network connection. Notify your manager if the problem persists.
ℹ
Do not use the browser’s own refresh (F5). Use only the in-app Refresh button. The browser refresh will reload the entire page and may briefly log out of the kitchen session.
8

Common Scenarios

The situations below come up regularly in a busy kitchen. Read through them so you know exactly what to do without having to stop and think during service.

A ticket appears but an item on it is 86’d (out of stock)
Do not advance the chip for the out-of-stock item. Call the floor immediately — a waiter must return to the table, inform the guest, and void the item from the Order Screen. Once voided, the chip will disappear from your KDS. If the chip remains and you are sure stock is zero, call your manager rather than marking the item Ready.
The KDS screen goes completely blank
  1. Wait 10 seconds — the 30-second auto-refresh may be mid-cycle.
  2. Tap the Refresh button in the toolbar.
  3. If blank persists, check network connectivity on the kitchen terminal (Wi-Fi or Ethernet indicator).
  4. If the network is fine, alert your manager. Continue using the back-up paper docket process until the system is restored.
An item was advanced by mistake (wrong chip tapped)
The KDS does not have an undo button. Notify your manager or a senior waiter immediately. The manager can void and re-send the item from the POS Order Screen, which will create a new chip at Sent (Blue) status. You then advance it normally when the item is actually ready.
A new ticket appears in a station column that does not handle that item
This typically means the menu item was assigned to the wrong station in the back office. Do not advance the chip. Alert your manager. They will update the item’s station assignment in the ERPNext Menu Item doctype and re-fire the order.
